# 02 搜索算法与问题求解

## 学习目标

- 理解状态空间搜索的基本思想。
- 掌握广度优先搜索、深度优先搜索和启发式搜索的区别。
- 理解 A* 算法中评价函数的作用。
- 能将简单问题建模为状态、动作和目标。

## 问题求解框架

许多人工智能问题可以抽象为状态空间搜索问题。系统从初始状态出发，通过一系列动作转移到新状态，最终到达目标状态。

一个搜索问题通常包含：

- 初始状态
- 可执行动作
- 状态转移规则
- 目标测试
- 路径代价

## 常见搜索算法

广度优先搜索会逐层扩展节点，适合寻找无权图中的最短路径，但空间开销较大。

深度优先搜索会沿一个方向尽可能深入，空间开销较小，但不一定找到最优解，也可能陷入很深的分支。

启发式搜索利用问题相关知识指导搜索方向。A* 算法使用：

```text
f(n) = g(n) + h(n)
```

其中 `g(n)` 表示从起点到当前节点的实际代价，`h(n)` 表示从当前节点到目标的估计代价。

## 示例

地图路径规划可以看作搜索问题：

- 状态：当前位置
- 动作：移动到相邻城市
- 目标：到达目的地
- 代价：距离或时间

如果启发函数设计合理，A* 算法可以更快找到较优路径。

## 易错点

- 广度优先搜索不等于启发式搜索。
- A* 的效果依赖启发函数质量。
- 启发式搜索不一定总是比无信息搜索更简单，关键在于启发函数是否合理。

# EduAgent 组长任务进展记录

## 本次完成的主线任务

本次工作聚焦在组长负责的“核心智能体策略与对话画像质量”部分，没有继续扩散到前端页面或后端数据库细节。

### 1. 新增画像事实抽取层

新增文件：

- `backend/app/services/profile_extractor.py`

作用：

- 从用户自然语言中稳定抽取学习画像字段。
- 解决“用户一句一句补充信息时，画像不能持续更新或误更新”的问题。
- 与大模型智能体配合使用，但不完全依赖大模型判断，避免每次对话结果不稳定。

当前支持抽取：

- 身份/专业背景：如“大一 学生 计算机”“我是软工大三”
- 目标课程：如“我想学数据结构”“不是机器学习，我要学数据结构”
- 已有基础：如“数据结构还可以”“线性代数还可以”
- 薄弱点：如“不会 PYTHON”“不会机器学习”“链表比较弱”
- 学习目标：如“为了考试通过”“入门”“做课程实验”
- 时间安排：如“48 小时完成”“两周”“每天 2 小时”
- 学习偏好：如“不要视频，多给我题和代码实验”
- 补充背景：如“男生”“运动员”等不会直接占用核心学习画像字段

### 2. 接入会话状态更新流程

修改文件：

- `backend/app/services/conversation_state.py`

作用：

- 用户每发一句话，系统会先进入会话状态；
- 会话状态调用画像抽取层，把可用事实写入当前 session；
- 后续画像页、学习路径、资源生成可以复用这些 session 事实。

这一步对应组员之前问的“智能体是不是只能用一次、下次对话不能更新”的问题：

- 当前设计不是一次性使用；
- 画像信息会随多轮对话持续更新；
- 后续如果接数据库持久化，这些 session facts 可以继续落库。

### 3. 新增画像抽取回归测试

新增文件：

- `backend/tests/profile_extractor_test.py`

新增覆盖场景：

- 短句身份碎片：`大一 学生 计算机`
- 丰富画像句子：专业、年级、强项、弱项一起出现
- 课程纠正：先说机器学习，后说不是机器学习要学数据结构
- 负向资源偏好：不要视频，只要题和代码实验
- 时间和目标拆分：48 小时完成神经网络入门
- 性别等低价值信息只进入补充背景，不占核心画像

### 4. 更新测试入口

修改文件：

- `backend/tests/run_conversation_tests.ps1`

作用：

- 把新增的画像抽取测试加入统一回归测试；
- 后续任何人改意图识别、画像抽取、会话状态，都可以一键跑这套测试，避免把已修好的对话能力改坏。

## 现在的能力边界

已经完成：

- 对话不再只靠固定回复；
- 用户多轮补充信息后，画像可以持续更新；
- 常见画像字段有确定性抽取规则；
- 性别、运动员等补充信息不会错误占用核心字段；
- 课程名中带“学习”“计算机”等词时，降低误判概率；
- 用户说 NLP、PYTHON、软工等简称时，画像显示尽量保留用户原话。

仍需后续继续做：

- 更多课程别名与知识点别名表；
- 数据库持久化后的画像版本记录；
- 资源库与课程知识库检索；
- 前端展示“本次画像更新了哪些字段”；
- Agent 之间互相调用与状态可视化；
- 多模态资源的缓存与生成进度。

## 给组员的接口提醒

前端侧：

- 不要写死画像数据；
- 尽量从后端 session 最新结果拿 profile/path/resources；
- 页面上最好显示“更新时间、数据来源、是否 AI 生成/规则抽取/模拟数据”。

后端侧：

- 后续数据库持久化时，优先保存 session、messages、facts、profile snapshot；
- 不要让数据库字段设计只适配当前 mock；
- AgentOrchestrator 调用前，应先读取当前 session facts。

智能体侧：

- 意图识别智能体只负责判断用户要干什么；
- 画像智能体/事实抽取层负责更新用户信息；
- 规划智能体负责生成学习路径；
- 资源智能体负责生成讲义、题目、导图、代码实验、视频脚本等资源；
- 评审智能体负责检查事实性、安全性和格式完整性。

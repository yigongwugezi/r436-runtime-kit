# EduAgent 项目定义文档

## 1. 项目名称

EduAgent：面向高校课程的多智能体个性化学习资源生成系统

## 2. 项目定位

本项目面向高等教育中的个性化学习需求，以《人工智能导论》课程为样例场景，构建一个能够通过自然语言对话理解学生情况、自动生成动态学习画像，并由多个智能体协同生成个性化学习资源、规划学习路径和提供智能辅导的学习智能体系统。

系统目标不是简单提供问答工具，而是形成“学生画像构建 - 知识短板诊断 - 学习路径规划 - 多模态资源生成 - 练习评估 - 动态优化”的闭环，帮助学生更高效地获取适配自身基础、目标和学习偏好的课程资源。

## 3. 样例课程选择

课程名称：《人工智能导论》

目标学生：

- 计算机、人工智能、电子信息、自动化等相关专业本科低年级学生
- 对人工智能感兴趣但基础参差不齐的跨专业学习者
- 具备基础 Python 编程能力或正在补齐编程基础的学生

课程范围：

- 人工智能概述与发展历史
- 搜索算法与问题求解
- 机器学习基础
- 神经网络基础
- 自然语言处理入门
- 计算机视觉入门
- 强化学习基础
- 人工智能伦理与安全

## 4. 核心业务问题

高校学生在学习专业课程时常见问题包括：

- 学习资源数量庞大但组织混乱，难以快速找到适合自身水平的材料
- 学生知识基础、学习目标、兴趣方向差异明显，统一教学难以兼顾个体需求
- 学生遇到知识盲点时，缺少持续、即时、个性化的辅导
- 传统资源以文字和课件为主，缺少视频、动画、实操案例、题库等多模态支持
- 学习效果反馈滞后，无法及时调整学习路径和资源推荐策略

EduAgent 通过多智能体协同机制，将学生画像、课程知识库、大模型推理、多模态内容生成和学习效果评估结合起来，提升学习资源的适配度和学习过程的可持续优化能力。

## 5. MVP 功能闭环

初赛阶段建议优先实现以下最小可演示闭环：

1. 学生通过自然语言对话输入个人情况，例如专业、基础、目标、学习困难和兴趣方向。
2. 系统自动抽取画像特征，生成不少于 6 个维度的动态学生画像。
3. 系统根据画像和课程知识库识别学生当前知识短板。
4. 多智能体协同生成个性化学习路径。
5. 系统生成至少 5 类学习资源：课程讲解文档、知识点思维导图、练习题、拓展阅读、代码实操案例。
6. 可选生成多模态视频脚本、动画分镜或教学 PPT，用于体现 SeeDance 等多模态生成技术融合思路。
7. 学生完成练习后，系统更新掌握情况，并调整后续学习路径和资源推荐。

## 6. 学生画像维度设计

系统通过对话式交互自动构建动态学生画像，初始设计包含以下维度：

| 画像维度 | 说明 | 示例 |
| --- | --- | --- |
| 专业背景 | 学生所属专业及相关课程经历 | 计算机专业大二，学过数据结构 |
| 知识基础 | 对课程先修知识的掌握程度 | Python 熟悉，概率论薄弱 |
| 学习目标 | 学生希望达成的学习结果 | 准备课程考试，完成 AI 项目 |
| 认知风格 | 偏好的学习方式 | 喜欢图解、案例、代码实验 |
| 易错点 | 常见薄弱环节和错误类型 | 混淆监督学习和无监督学习 |
| 编程能力 | 实操和代码理解能力 | 能读懂基础代码，但独立实现困难 |
| 学习进度 | 当前课程学习位置 | 正在学习神经网络基础 |
| 兴趣方向 | 对具体技术方向的兴趣 | 偏好自然语言处理和大模型应用 |

画像应支持“随学随新”：当学生完成练习、提问或反馈资源难度后，系统更新其知识掌握情况和推荐策略。

## 7. 多智能体架构设计

系统采用多智能体协同架构，各智能体职责如下：

| 智能体名称 | 核心职责 | 主要输出 |
| --- | --- | --- |
| 对话画像智能体 | 从自然语言对话中提取学生特征，构建动态画像 | 学生画像 JSON、画像摘要 |
| 课程知识库智能体 | 检索课程文档、知识点、案例和题库素材 | 相关知识片段、参考来源 |
| 学习诊断智能体 | 分析知识短板、学习难度和优先级 | 知识短板报告 |
| 路径规划智能体 | 根据画像和诊断结果生成学习路径 | 阶段化学习计划 |
| 内容讲解智能体 | 生成个性化课程讲解文档 | Markdown 讲义 |
| 题库生成智能体 | 生成选择题、判断题、简答题、代码题 | 个性化练习题 |
| 思维导图智能体 | 组织知识结构和概念关系 | Mermaid 或 Markdown 思维导图 |
| 实操案例智能体 | 生成代码实验和实践项目材料 | Python 案例、实验步骤 |
| 多模态资源智能体 | 生成 PPT 大纲、视频脚本、动画分镜 | 多模态资源设计方案 |
| 评估反馈智能体 | 分析练习结果和学习反馈，调整画像 | 评估报告、更新建议 |

其中，初赛 MVP 至少实现前 8 个智能体的核心流程，多模态资源智能体和评估反馈智能体可作为加分亮点。

## 8. 个性化资源类型

系统至少生成以下资源：

1. 个性化课程讲解文档：根据学生基础调整讲解深度和表达方式。
2. 知识点思维导图：帮助学生建立章节结构和概念关系。
3. 个性化练习题：包含基础巩固题、概念辨析题、代码实操题和综合应用题。
4. 拓展阅读材料：推荐论文、博客、教材章节或开源项目，并说明适合原因。
5. 代码类实操案例：以 Python 为主，生成可运行实验步骤和代码片段。
6. 多模态教学资源：生成 PPT 大纲、短视频脚本、动画分镜或 SeeDance 视频生成提示词。

## 9. 技术路线建议

前端：

- Web 应用优先，便于演示和部署
- 推荐使用 React、Vue 或 Streamlit 快速构建
- 支持聊天式交互、Markdown 渲染、资源卡片、生成进度展示

后端：

- 推荐使用 Python FastAPI 或 Node.js
- 提供画像构建、智能体调度、知识库检索、资源生成等接口

智能体框架：

- 可使用 LangGraph、AutoGen、CrewAI 或自研轻量编排框架
- 重点展示“多角色协同、任务拆解、结果汇总、质量检查”的流程

知识库：

- 自建《人工智能导论》课程文档集
- 可使用 Markdown、PDF、教材摘要、课程讲义作为初始输入
- 可选向量数据库：FAISS、Chroma、Milvus

大模型与 AI 工具：

- 通用大模型用于对话、画像抽取、内容生成和评估
- 多模态生成模型用于视频脚本、动画分镜、教学素材生成
- AI Coding 工具用于辅助开发，并在提交文档中说明使用方式
- 若赛题要求使用科大讯飞相关工具，应在实际技术选型中优先接入讯飞星火、讯飞开放平台能力或相关工具链

## 10. 防幻觉与内容安全机制

系统需设计以下机制：

- 知识库增强生成：课程相关内容优先基于本地知识库检索结果生成
- 引用来源标注：关键知识点、定义和公式标注参考来源
- 多智能体互审：由质量检查智能体检查内容准确性、难度匹配和敏感风险
- 不确定性提示：当知识库缺少证据时，提示“当前资料不足”，避免编造
- 安全过滤：过滤违规、敏感、学术不端和不适合学生的内容
- 教育边界声明：系统提供学习辅导，不直接代写作业或替代学生完成考试

## 11. 演示流程设计

建议 7 分钟演示视频采用以下结构：

| 时间 | 内容 |
| --- | --- |
| 0:00 - 0:40 | 介绍高校个性化学习痛点和项目目标 |
| 0:40 - 1:20 | 展示系统首页和对话式画像构建 |
| 1:20 - 2:20 | 展示学生画像、知识短板诊断结果 |
| 2:20 - 3:30 | 展示多智能体协同生成学习路径 |
| 3:30 - 5:20 | 展示讲义、思维导图、题库、阅读材料、代码案例等资源 |
| 5:20 - 6:20 | 展示智能答疑或学习效果评估加分功能 |
| 6:20 - 7:00 | 总结创新价值、技术架构和应用前景 |

## 12. 初赛交付物规划

初赛建议提交以下内容：

- 演示 PPT：展示应用价值、技术方案、系统架构、核心功能、创新点和演示截图
- 项目源码：包含前端、后端、智能体编排、知识库样例、配置文件
- 课程知识库：至少一门完整高校专业课程的文档集
- 演示视频：7 分钟内展示完整操作流程
- 系统开发说明书：说明需求分析、架构设计、智能体设计、功能实现、部署方式
- 测试说明书：说明功能测试、性能测试、内容安全测试和典型案例
- AI Coding 工具说明：说明使用的 AI 编程工具、用途和人工审核机制
- 开源协议说明：列出使用的开源框架、来源和协议

## 13. 第一阶段开发任务

第一阶段建议完成以下任务：

1. 整理《人工智能导论》课程知识库大纲。
2. 设计学生画像 JSON Schema。
3. 设计多智能体协作流程图。
4. 搭建 Web 原型页面，包括对话区、画像区、资源区、学习路径区。
5. 实现画像抽取和学习路径生成的基础接口。
6. 实现 5 类资源生成模板。
7. 准备 1 个完整演示案例。

## 14. 推荐演示案例

学生输入：

“我是电子信息专业大二学生，学过 Python，但机器学习基础比较薄弱。我想用两周时间入门人工智能，重点理解神经网络和自然语言处理，希望多给我一些图解、代码案例和练习题。”

系统输出：

- 学生画像：电子信息专业、Python 基础中等、机器学习薄弱、目标为两周入门 AI、偏好图解和代码案例
- 知识短板：概率统计、机器学习基本概念、神经网络训练流程
- 学习路径：第 1 阶段补基础，第 2 阶段学机器学习，第 3 阶段学神经网络，第 4 阶段做 NLP 小项目
- 学习资源：讲义、思维导图、题库、拓展阅读、Python 实验、视频脚本
- 反馈机制：根据练习错题更新易错点，并推送补充讲解

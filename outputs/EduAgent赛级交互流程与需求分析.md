## 1. 需求分析总览

### 1.1 业务目标

构建一个面向高校课程的**多智能体个性化学习资源生成系统**，实现从「自然语言对话」到「全链路个性化学习方案」的端到端闭环。

### 1.2 赛题关键要求映射

| 赛题要求 | EduAgent 实现 | 状态 |
|---------|--------------|------|
| 大模型应用 | 多智能体协同（Profile / Diagnosis / Planner / Resource / Review） | ✅ 已实现 |
| 个性化学习资源生成 | 5 类资源：讲义、思维导图、练习题、拓展阅读、实操案例 | ✅ 已实现 |
| 自适应学习 | 画像维度驱动 → 诊断短板 → 规划路径 → 动态调整 | ✅ 已实现 |
| 多模态输出 | Markdown 讲义 + Mermaid 思维导图 + 代码案例 + 题库 | ✅ 已实现 |
| 知识库增强 | 基于本地课程知识库检索内容，减少大模型幻觉 | 🟡 部分实现 |
| 学习效果追踪 | 学习事件埋点 + 掌握度更新 + 弱项分析 | ✅ 已实现 |
| 防幻觉与内容安全 | ReviewAgent 质量检查 + 不确定性提示 | 🟡 需要强化 |

### 1.3 核心数据流（闭环）

```
用户输入 → 画像抽取 → 知识检索 → 诊断分析 → 路径规划 → 资源生成 → 质量检查 → 结果展示
                                                                          ↓
用户反馈 ← 学习效果追踪 ← 练习评估 ← 资源消费 ← 开始学习 ←──────────────
```

## 2. 用户角色与场景定义

### 2.1 角色

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **高校学生（主角色）** | 计算机/AI/电子信息等专业本科生 | 获得适合自己基础和目标的学习方案 |
| **课程教师（辅助角色）** | 课程负责人或助教 | 查看班级学习情况，调整教学策略 |
| **评委/演示观众** | 大赛评审 | 直观理解技术方案创新性和完成度 |

### 2.2 典型使用场景

#### 场景 A：首次使用——自然语言构建画像并生成方案

> **用户输入**："我是电子信息专业大二学生，学过 Python，但机器学习基础比较薄弱。我想用两周时间入门人工智能，重点理解神经网络和自然语言处理，希望多给我一些图解、代码案例和练习题。"

**预期输出**：
- ✅ 10 维度学生画像（含置信度）
- ✅ 知识短板诊断报告（含优先级）
- ✅ 14 天分阶段学习路径
- ✅ 5 类个性化学习资源

#### 场景 B：继续学习——查看已有画像和路径

> 用户第二次访问时，系统自动加载历史会话，展示之前生成的画像、路径和资源，无需重复输入。

#### 场景 C：完成练习后系统更新

> 用户完成一套练习题 → 前端上报学习事件 → 后端更新掌握度 → 路径节点状态自动变更（未掌握 → 学习中 → 已掌握）

#### 场景 D：调整学习目标

> 用户说"我觉得两周太赶了，改成一个月" → 系统增量更新画像 → 重新规划路径 → 保留已有资源，生成新增资源

## 3. 核心交互流程（完整闭环）

### 3.1 总览流程图

```mermaid
graph TB
    subgraph "用户侧"
        A[打开首页] --> B[点击"开始学习"或"AI 对话"]
        B --> C[输入或选择快捷指令]
    end

    subgraph "系统响应层"
        C --> D{是否有历史会话?}
        D -->|是| E[加载历史会话数据]
        D -->|否| F[创建新会话]
        E --> G[显示对话页面]
        F --> G
    end

    subgraph "多智能体处理层"
        G --> H[发送消息到后端]
        H --> I[IntentAgent 意图分类]
        I --> J{意图类型?}
        J -->|学习规划| K[启动全链路智能体流程]
        J -->|闲聊| L[直接回复]
        J -->|日期/时间| M[系统信息回复]
        J -->|澄清| N[追问补充信息]

        K --> O[ProfileAgent: 构建画像]
        O --> P[KnowledgeAgent: 检索知识库]
        P --> Q[DiagnosisAgent: 诊断短板]
        Q --> R[PlannerAgent: 规划路径]
        R --> S[ResourceAgent: 生成资源]
        S --> T[ReviewAgent: 质量检查]
        T --> U[统一结果封装]
    end

    subgraph "前端展示层"
        U --> V[流式推送进度]
        V --> W[聊天区展示方案摘要]
        W --> X[自动更新画像页]
        X --> Y[自动更新路径页]
        Y --> Z[自动更新资源库]
    end

    subgraph "持续优化层"
        Z --> AA[用户学习或提问]
        AA --> AB[上报学习事件]
        AB --> AC[更新掌握度]
        AC --> AD[路径节点状态刷新]
        AD --> AE[推荐下一阶段资源]
        AE --> AA
    end
```

### 3.2 详细步骤说明

| 步骤 | 触发 | 前端行为 | 后端行为 | 预期反馈 |
|------|------|---------|---------|---------|
| 1. 进入首页 | 用户打开系统 | 展示首页（功能简介 + 统计 + 特色卡片） | 无 | 视觉吸引 + 快速了解系统 |
| 2. 启动对话 | 点击「AI 对话」或「开始学习」 | 路由到 `/chat`，显示空对话页 + 快捷指令 | 无 | 友好的对话启动引导 |
| 3. 输入/选择指令 | 用户打字或点快捷指令 | 消息发送 → 显示用户气泡 → 显示生成进度条 | 创建/恢复 session | 看到消息已发送 |
| 4. 意图识别 | 消息到达后端 | 显示「正在分析你的需求…」 | IntentAgent 分类 | 告知用户系统正在处理 |
| 5. 全链路处理 | 意图为「学习规划」 | 实时进度：画像构建 → 知识检索 → 诊断分析 → 路径规划 → 资源生成 → 质量检查 | 6 个 Agent 顺序执行 | 进度条逐步推进，增强信任感 |
| 6. 结果展示 | 全流程完成 | 聊天区显示方案摘要 → 侧边栏更新画像/路径/资源 | 存储结果到 conversation_store | 多页面数据自动同步 |
| 7. 详情查看 | 用户切换到其他页面 | 各页面从后端获取最新数据 | 返回已有数据 | 画像页显示雷达图 + 维度详情 |
| 8. 持续交互 | 用户继续提问 | 上下文关联，支持追问和调整 | 增量更新画像和路径 | 学习目标调整后路径自动重建 |

### 3.3 流式进度推送机制

```
消息发送
  │
  ├──▶ [progress: 10%] "正在构建学习画像…"         ← ProfileAgent
  ├──▶ [progress: 25%] "正在检索课程知识点…"        ← KnowledgeAgent
  ├──▶ [progress: 40%] "正在诊断知识短板…"          ← DiagnosisAgent
  ├──▶ [progress: 55%] "正在规划学习路径…"          ← PlannerAgent
  ├──▶ [progress: 70%] "正在生成学习资源（讲义、思维导图…）" ← ResourceAgent
  ├──▶ [progress: 85%] "正在进行质量检查…"          ← ReviewAgent
  ├──▶ [progress: 95%] "正在生成最终方案…"
  └──▶ [progress: 100%] "方案已生成，点击查看详情"
```

## 4. 页面级交互设计

### 4.1 页面拓扑

```
/ (首页)
├── /chat (AI 对话 — 核心入口)
├── /profile (学习画像 — 自动同步)
├── /path (学习路径 — 自动同步)
├── /resources (资源库 — 自动同步)
└── /resources/:id (资源详情)
```

### 4.2 各页面详细交互

#### 4.2.1 首页 `/`

**功能**：系统总览入口，吸引用户了解功能并开始使用

**元素**：
- 顶部 Hero：系统名称 + 一句话定位 + CTA 按钮「开始学习」
- 数据统计：服务学生数、效率提升、用户评分等
- 核心功能卡片（4 张）：多智能体架构 / 10 维度画像 / 动态路径 / 多模态资源
- 三步使用引导：对话描述 → 智能分析 → 个性学习
- 智能体矩阵展示：展示各 Agent 图标、名称和职责
- 用户评价轮播
- 底部页脚

**交互设计**：
- 点击「开始学习」→ 导航到 `/chat`
- 功能卡片 hover 有阴影和上浮动效
- 三步引导区支持 scroll-trigger 动画

#### 4.2.2 AI 对话页 `/chat`

**功能**：核心交互入口，所有用户输入和 AI 回复在此完成

**元素**：
- 消息列表（气泡式）
- 用户消息：灰色背景 + 右对齐 + 头像 U
- AI 消息：白色背景 + 左对齐 + 品牌色头像
- 生成进度指示器（Agent 步骤轮播 + 进度条）
- 消息输入框（支持多行 + Enter 发送 + Shift+Enter 换行）
- 快捷指令按钮（6 个预设）
- 停止生成按钮（流式传输中显示）
- 滚动至底按钮（不在底部时显示）

**交互设计**：
- **消息渲染**：AI 回复支持 Markdown（标题/列表/代码块/表格）+ Mermaid 图表 + LaTeX 公式
- **进度展示**：Agent 执行期间显示步骤轮播（每步 2-3 秒切换 + 进度条百分比）
- **快捷指令**：首次打开时显示，发送消息后自动隐藏
- **自动滚动**：新消息到达时自动滚至底部，用户手动上滑浏览历史时暂停自动滚动
- **停止生成**：点击后立即中止流式请求，保持已收到内容
- **空状态**：无消息时居中显示品牌标识和引导文案

**状态管理**：
```
chatStore:
  - sessions: ChatSession[]      // 会话列表
  - currentSessionId: string     // 当前会话 ID
  - messages: ChatMessage[]      // 当前会话消息
  - isStreaming: boolean         // 是否正在流式接收
  - agentProgress: {             // 智能体进度
      stage: string,
      progress: number
    } | null
  - error: string | null         // 错误信息
```

#### 4.2.3 学习画像页 `/profile`

**功能**：可视化展示当前会话的学生画像

**元素**：
- 画像总览卡片：姓名/年级/学习天数/练习准确率
- 雷达图：各维度评分可视化（使用 Chart.js / Recharts）
- 维度详情列表：每个维度显示标签、值、置信度、更新时间
- 知识短板区：显示诊断出的薄弱知识点 + 优先级别
- 学习偏好卡片：偏好格式、学习节奏、难度等级
- 学习历史卡片：总学习时长、连续学习天数、最近学习时间

**交互设计**：
- 雷达图支持 hover 显示维度详情
- 短板卡片支持点击 → 导航到资源页查看对应资源
- 空状态：显示「尚未生成画像，请先在 AI 对话中描述你的学习需求」
- 自动刷新：从 `/chat` 页面生成方案后，切换到本页自动展示最新数据

**数据流**：
```
GET /api/profile?sessionId=xxx
  → 返回 { profile: StudentProfile }
  → StudentProfile 包含 dimensions[], weaknesses[], preferences[], history[]
```

#### 4.2.4 学习路径页 `/path`

**功能**：展示个性化分阶段学习路径

**元素**：
- 路径总览卡：总天数、阶段数、总进度
- 时间线式阶段列表（每个阶段一个可展开卡片）
- 阶段卡片：阶段标题、目标描述、预计天数、节点列表
- 每个节点：知识点名称、掌握度进度条、状态（locked/available/in_progress/completed）
- 节点展开：关联资源列表 + 资源类型标签 + 是否核心节点

**交互设计**：
- 阶段卡片默认仅展开第 1 个，其余折叠
- 掌握度进度条颜色渐变：红(<30%) → 黄(30-60%) → 绿(>60%)
- 节点状态用图标区分：🔒 locked / 🔓 available / ⏳ in_progress / ✅ completed
- 点击资源 → 导航到资源详情页
- 路径图使用垂直时间线布局，清晰展示学习顺序

**数据流**：
```
GET /api/learning-path?sessionId=xxx
  → 返回 { path: LearningPath }
  → LearningPath 包含 stages[], 每阶段含 nodes[]
```

#### 4.2.5 资源库页 `/resources`

**功能**：展示所有生成的个性化学习资源

**元素**：
- 资源类型筛选标签（全部 / 讲义 / 思维导图 / 练习题 / 拓展阅读 / 实操案例）
- 资源卡片网格布局
- 每张卡片：类型图标、标题、描述、知识点标签、难度、预计时长、收藏状态
- 资源详情弹窗/页面：完整内容渲染（Markdown + Mermaid + 代码高亮）

**交互设计**：
- 筛选标签：点击切换类型，卡片列表带过渡动画
- 资源卡片 hover 效果：上浮 + 阴影加深
- 点击收藏：图标切换（空心 → 实心），后端保存状态
- 点击卡片：打开资源详情，支持完整内容阅读
- 空状态：分两种情况
  - 无会话：提示「请先在 AI 对话中生成学习方案」
  - 有会话无资源：提示「正在生成资源，请稍候…」

**数据流**：
```
GET /api/resources?sessionId=xxx&type=lecture
  → 返回 { resources: Resource[], total, page }

GET /api/resources/:id?sessionId=xxx
  → 返回 { resource: Resource }
```

### 4.3 关键交互细节

#### 4.3.1 多页面数据同步

```
[chat] 用户发送消息 → 后端处理完成
  │
  ├─▶ chatStore 更新 messages + agentProgress
  │
  ├─▶ 画像页 (profileStore) 在 mount 时 GET /api/profile?sessionId=xxx
  ├─▶ 路径页 (learningPathStore) 在 mount 时 GET /api/learning-path?sessionId=xxx
  └─▶ 资源页 (resourceStore) 在 mount 时 GET /api/resources?sessionId=xxx
```

**注意**：不在聊天页主动推送到其他 store，保持各页面独立获取，避免状态耦合。但聊天页生成完成后，底部导航栏对应的图标可以加一个小红点或呼吸动效提示「有新数据」。

#### 4.3.2 会话持久化

```
- localStorage 存储会话 ID 列表
- 切换到已有会话时 POST /chat/sessions/:id/reset 恢复上下文
- 新建会话生成新 UUID → 存入 localStorage
- 最多保留 10 个历史会话
```

#### 4.3.3 加载与错误状态

| 状态 | 表现 |
|------|------|
| 加载中 | Skeleton 骨架屏（消息气泡 / 卡片 / 列表） |
| 加载失败 | 错误提示 + 重试按钮 |
| 空数据 | 友好提示 + 引导操作（如「去聊天页生成方案」） |
| 超时 | `120s` timeout，显示「请求超时，请重试」 |
| 网络离线 | 检测 navigator.onLine，显示离线提示 |

## 5. API 契约与数据流

### 5.1 接口总览

| 模块 | 方法 | 路径 | 用途 | 数据流向 |
|------|------|------|------|---------|
| 聊天 | POST | `/api/chat/stream` | 流式对话 | 后端 → SSE → 前端 |
| 聊天 | POST | `/api/chat/send` | 非流式对话 | 前端 ↔ 后端 |
| 会话 | GET | `/api/chat/sessions` | 会话列表 | 后端 → 前端 |
| 会话 | POST | `/api/chat/sessions/{id}/reset` | 切换/重置会话 | 前端 → 后端 |
| 进度 | GET | `/api/chat/progress/{task_id}` | 轮询生成进度 | 后端 → 前端 |
| 指令 | GET | `/api/chat/quick-commands` | 快捷指令列表 | 后端 → 前端 |
| 画像 | GET | `/api/profile?sessionId=xxx` | 获取画像 | 后端 → 前端 |
| 画像 | POST | `/api/profile/build` | 构建画像 | 前端 → 后端 |
| 画像 | PATCH | `/api/profile` | 更新画像 | 前端 → 后端 |
| 路径 | GET | `/api/learning-path?sessionId=xxx` | 获取路径 | 后端 → 前端 |
| 路径 | POST | `/api/learning-path/generate` | 生成路径 | 前端 → 后端 |
| 路径 | PATCH | `/api/learning-path/nodes/{id}` | 更新节点掌握度 | 前端 → 后端 |
| 资源 | GET | `/api/resources?sessionId=xxx` | 资源列表 | 后端 → 前端 |
| 资源 | GET | `/api/resources/{id}` | 资源详情 | 后端 → 前端 |
| 资源 | POST | `/api/resources/generate` | 生成资源 | 前端 → 后端 |
| 资源 | POST | `/api/resources/{id}/bookmark` | 收藏/取消收藏 | 前端 → 后端 |
| 资源 | GET | `/api/resources/{id}/knowledge-graph` | 知识图谱 | 后端 → 前端 |
| 反馈 | POST | `/api/feedback` | 提交评分反馈 | 前端 → 后端 |
| 反馈 | POST | `/api/feedback/event` | 上报学习事件 | 前端 → 后端 |
| 分析 | GET | `/api/learning-analytics` | 学习分析 | 后端 → 前端 |

### 5.2 前后端协作流程（流式场景）

```
前端                              后端
  │                                 │
  │  POST /api/chat/stream          │
  │  { sessionId, message }         │
  │ ──────────────────────────────▶ │
  │                                 │ IntentAgent 分类
  │  SSE: data: {"stage":"画像构建","progress":10}
  │ ◀────────────────────────────── │ ProfileAgent
  │  SSE: data: {"stage":"知识检索","progress":25}
  │ ◀────────────────────────────── │ KnowledgeAgent
  │  SSE: data: {"stage":"诊断分析","progress":40}
  │ ◀────────────────────────────── │ DiagnosisAgent
  │  SSE: data: {"stage":"路径规划","progress":55}
  │ ◀────────────────────────────── │ PlannerAgent
  │  SSE: data: {"stage":"资源生成","progress":70}
```

## 6. 状态管理与多会话支持

### 6.1 前端状态分层

```
Zustand Stores:
├── chatStore
│   ├── sessions: ChatSession[]          // 会话列表
│   ├── currentSessionId: string | null  // 当前会话
│   ├── messages: ChatMessage[]          // 当前消息
│   ├── isStreaming: boolean
│   ├── agentProgress: { stage, progress } | null
│   └── error: string | null
│
├── profileStore
│   ├── profile: StudentProfile | null
│   ├── loading: boolean
│   └── error: string | null
│
├── learningPathStore (useLearningPath hook 内部)
│   ├── path: LearningPath | null
│   ├── loading: boolean
│   └── error: string | null
│
└── resourceStore (useResources hook 内部)
    ├── resources: Resource[]
    ├── total: number
    ├── page: number
    ├── loading: boolean
    └── error: string | null
```

### 6.2 多会话协作

```
用户切换会话:
1. chatStore.setCurrentSession(newSessionId)
2. 聊天页: 使用新 sessionId 请求历史消息
3. 画像页: profileStore.fetch() 自动使用 chatStore.currentSessionId
4. 路径页: learningPathStore.fetch() 自动使用 chatStore.currentSessionId
5. 资源页: resourceStore.fetch() 自动使用 chatStore.currentSessionId

关键设计:
- profileStore 不缓存 profile，每次 mount 重新获取
- learningPathStore 不缓存 path，每次 mount 重新获取
- resourceStore 不缓存 resources，每次 mount 重新获取
→ 保证"切换会话后所有页面自动同步到新会话数据"
```

## 7. 错误处理与边界情况

### 7.1 错误分类与处理策略

| 错误类型 | 示例 | 前端响应 | 后端响应 |
|---------|------|---------|---------|
| 输入为空 | 用户发送空消息 | 前端拦截，提示"请输入内容" | — |
| 输入过短 | 用户只发"你好" | 正常发送 | IntentAgent 识别为闲聊，友好回复 |
| 输入过长 | 超过 4000 字符 | 前端限制，显示计数 | — |
| 会话不存在 | sessionId 过期或无效 | 自动创建新会话 | 返回 40003，前端新建后重试 |
| 智能体超时 | Agent 执行超过 60s | 显示"生成超时，请重试" | 终止当前流程，返回 40004 |
| LLM 不可用 | API Key 无效或网络不通 | 显示"AI 服务暂时不可用" | 返回 50001，fallback 到 Mock |
| 资源生成失败 | 某类资源生成出错 | 跳过该类，展示其他资源 | ReviewAgent 标记后继续 |
| 并发请求 | 用户在上次未完成时再次发送 | 显示"正在生成中，请稍候" | 忽略新请求或返回 busy 状态 |
| 网络断连 | 浏览器离线 | 显示离线提示条，恢复后自动重试 | — |

### 7.2 边界情况处理清单

- [ ] **空会话**：无历史会话时显示引导文案，而不是空白页
- [ ] **空画像**：未生成画像时，画像页显示「请先描述学习需求」
- [ ] **空路径**：未生成路径时，路径页显示引导
- [ ] **空资源**：未生成资源时，资源库显示空状态
- [ ] **重复输入**：用户连续发送相同消息 → 正常处理，不做幂等
- [ ] **快速切换页面**：流式请求未完成时切换页面 → 不中断请求，后台继续
- [ ] **多 Tab 操作**：同一用户多 Tab 访问 → 各自独立，不共享状态
- [ ] **大内容渲染**：AI 回复包含超长 Markdown → 代码块折叠、表格横向滚动
- [ ] **Mermaid 渲染失败**：语法错误时 → 显示原始代码 + 重试按钮
- [ ] **刷新页面**：刷新后保留 sessionId，恢复消息列表
